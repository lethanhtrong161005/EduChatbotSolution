# AI Coding & XML Documentation Standard

This document defines the standard structure for source code and XML Documentation within the **EduChatAI** project. The AI must read and strictly adhere to 100% of the rules below when creating or updating code.

---

## 1. AI Instructions

When requested to write C# source code (Classes, Interfaces, Methods, Properties) by the user, the AI must:
1. **Automatically generate XML Documentation** in **English**.
2. Use all standard tags completely: `<summary>`, `<param>`, `<returns>`, and `<exception>` (if error-handling logic exists).
3. Keep sentences short, clear, and focused on explaining business logic, rather than generic programming language syntax.
4. Adhere to modern C# standards (C# 12/13), utilizing `File-scoped namespaces`, nullable attributes (`?`), and default string initialization (`= string.Empty`).

---

## 2. Reference Template

The AI must precisely mimic the comment style and code organization demonstrated in the reference class below:

```csharp
using System;
using System.Collections.Generic;
using System.Threading.Tasks;

namespace EduChatAI.Web.Docs
{
    /// <summary>
    /// Reference class defining the standard for code writing and automatic documentation generation.
    /// Every class generated by the AI must include this high-level overview.
    /// </summary>
    public class AiDocStandard
    {
        /// <summary>
        /// The unique identifier of the reference object in the system.
        /// </summary>
        public int Id { get; set; }

        /// <summary>
        /// The title of the document or text to be processed by the system.
        /// </summary>
        public string Title { get; set; } = string.Empty;

        /// <summary>
        /// The current status of the processing pipeline (e.g., Pending, Processing, Completed).
        /// </summary>
        public string Status { get; set; } = "Pending";

        /// <summary>
        /// Splits the text into smaller chunks to prepare for Embedding Vector generation.
        /// </summary>
        /// <param name="documentContent">The full raw text content to be split.</param>
        /// <param name="maxChunkSize">The maximum number of characters per chunk (Default is 500 characters).</param>
        /// <returns>A list of successfully fragmented text strings.</returns>
        /// <exception cref="ArgumentNullException">Thrown when the input documentContent is null or whitespace-only.</exception>
        /// <exception cref="ArgumentOutOfRangeException">Thrown when the configured maxChunkSize is less than or equal to 0.</exception>
        /// <remarks>
        /// This method utilizes a trailing punctuation algorithm to preserve sentence context.
        /// </remarks>
        public async Task<List<string>> SplitDocumentIntoChunksAsync(string documentContent, int maxChunkSize = 500)
        {
            // 1. Input validation
            if (string.IsNullOrWhiteSpace(documentContent))
            {
                throw new ArgumentNullException(nameof(documentContent), "Document content cannot be empty.");
            }

            if (maxChunkSize <= 0)
            {
                throw new ArgumentOutOfRangeException(nameof(maxChunkSize), "Chunk size must be greater than 0.");
            }

            // 2. Asynchronous business logic processing
            return await Task.Run(() =>
            {
                var chunks = new List<string> 
                { 
                    "Sample text chunk 1", 
                    "Sample text chunk 2" 
                };
                return chunks;
            });
        }
    }
}
